본인의 정체성·핵심 정의·서비스 본질 같은 *근본 단어*를 박을 때는 일방적으로 결정하지 않는다. 후보를 펼치고, 각 후보의 trade-off를 보여주고, *사용자가 박는다*. "박았다"·"확정했다"라는 표현은 사용자가 명시적으로 동의한 후에만 사용.

**Why:** 2026-05-23 윤소정 사이트 개편 작업 중, 사용자가 "프레임 파는 사람"이라고 자기를 언급한 걸 내가 *프레임/프레임워크/그림*까지 임의로 확장해 정의로 박았다고 표현. 사용자가 *합의 누락*을 지적: "정의를 박은 적 없어. 그건 네가 정한 거지 나랑 합의를 해야지."

**How to apply:** 
1. 본질·정체성·서비스 정의·핵심 단어는 후보 형식으로만 제시
2. "박혔다"·"정해졌다"라는 단정 표현은 사용자가 *명시적으로 "OK 박자"*라고 말한 후에만
3. 사용자의 *요청*("그렇게 정리해줘")과 *결정*("이걸로 박자")을 구분
4. 정의 결정 단계에서는 항상 사용자에게 박을 권한을 명시적으로 넘김
5. 관련: [[user-profile]] — 사용자는 본인 콘텐츠/정체성에 대한 *판단 주체*이고 AI는 *분석·검증 보조*
