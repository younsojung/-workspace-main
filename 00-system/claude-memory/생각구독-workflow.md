생각구독 발행 워크플로우 8단계 — 81호(슈퍼샘플의 시대)에서 정립.

1. **한 달 정리** — Raw Material 인터뷰 (사람·사건·성과 3축 10건, 4질문)
2. **핵심 개념 추출** — 10건의 공통점 한 단어
3. **컨셉 + 부제 + 키 퀘스천** — 카피라이터 톤 부제, 머릿속 그림
4. **표지 무드보드 + 디자이너 브리프** — `cover_brief.md`
5. **이야기 설계** — 큰 아크 + 첫·마지막 챕터 + 80편 코퍼스 참조
6. **목차** — INTRO + 챕터 3~5개 + CLOSING. `toc.html`로 같이 체크
7. **본문 작성** — 챕터별 사이클 (raw → 정리체 → 채팅창 → toc.html 미리보기 → OK → body_NN.md)
8. **구글 문서 이전** — `full_manuscript.md` + `export.html` (전체 복사 버튼)

**Why:** 사용자가 81호 작업 중 "앞으로 생각구독 작업할 때 우리가 작업했던 순서를 기억해서 작업해야 해. 한 달을 정리하고 컨셉을 잡고 키퀘스천 잡고... 기억나니?"라고 명시. 8단계 순서가 *공식 표준*임을 박음.

**How to apply:**
1. 사용자가 "이번달 호 시작" / "생각구독 N호" 등으로 진입하면 *반드시 1단계부터* 시작 (이미 진행된 호면 progress.md 확인 후 그 단계부터 이어감)
2. 8단계 외 추가 작업 (팩트체크·코퍼스 리서치)은 *해당 단계 안*에서 처리
3. 본문체: INTRO·CLOSING = 편지체 / 챕터 = 정리체 (~다)
4. 정의·정체성 단어는 *사용자가 박는다* (합의 누락 ❌)
5. 외부 공개 숫자 가드 — 활성 구독자·매출 등 카피 X
6. 상세 워크플로우: `30-knowledge/생각구독-corpus/_aggregations/workflow-from-81.md`
