# 주간 캠페인·채널 통합 보고서 템플릿

> 러쉬코리아 브랜드 커뮤니케이션 기획팀이 매주 월요일 본부장에게 올리는 1페이지 보고서.
> 매번 같은 양식으로 나오게 박아두는 용도.

## 입력 데이터

| 파일 | 역할 |
|---|---|
| `channel_sales.csv` | 채널별 일매출 (자사앱·자사몰·무신사·29CM·SSG·갤러리아) |
| `instagram_posts.csv` | @lushkorea 게시물 성과 (캠페인 태그로 매출과 연결) |
| `campaign_calendar.csv` | 캠페인 마스터 (예산·기간·타깃 KPI) |
| `app_acquisition.csv` | 자사앱 신규 다운로드 + 러쉬 어스 멤버십 가입 |

## 분석 윈도

- **이번 주**: 보고서 생성 시점 기준 직전 7일
- **비교 기준**: 직전 7일 (지지난 주)
- 데이터 마지막 날짜를 "오늘"로 간주하고 7일씩 잘라낸다.

## 보고서 출력 구조 (4섹션, A4 1~2장)

### 1. 한 줄 요약 (헤더)
```
[YYYY-MM-DD ~ YYYY-MM-DD] 부문 매출 X.XX억 / 직전 주 대비 ±N% / 가장 큰 변화: {한 줄}
```

### 2. 채널별 표 (6행)
컬럼: 채널 / 이번주 매출 / 직전주 매출 / 증감률 / 주문수 / AOV / 환불률 / 알림
- 채널 6개 전부 (자사앱·자사몰·무신사·29CM·SSG·갤러리아)
- 정렬: 매출 내림차순

### 3. 캠페인 ROI 알림 (이번 주 활성 캠페인만)
컬럼: 캠페인명 / 기간 / 예산 사용률 / 인스타 도달 합 / 추정 기여 매출 / ROI / 판정
- 추정 기여 매출 = 캠페인 태그 게시물의 reach 가중치 × 자사앱+자사몰 spike 매출
- ROI = 추정 기여 매출 / (예산 × 기간 비율)
- 활성 캠페인이 5개를 넘으면 ROI 낮은 순으로 상위 5개만

### 4. 의사결정 후보 (3개 이내)
형식: 각 항목마다 (배경 한 줄 / 데이터 근거 한 줄 / 선택지 2~3개 / 본부장 의견 자리 빈 칸)
- 데이터에서 발견된 이상 신호 중 임원 회의 안건이 될 만한 것
- 3개를 넘으면 가장 임팩트 큰 3개만 (매출 임팩트 또는 캠페인 예산 임팩트 기준)

## 검출 규칙 (이상 신호)

| 신호 | 기준 | 색상 |
|---|---|---|
| 채널 매출 급변 | 이번주가 직전주 대비 ±20% 이상 | 노랑 |
| 채널 매출 급락 | 이번주가 직전주 대비 -30% 이하 | 빨강 |
| 캠페인 ROI 미달 | ROI < 1.0 (예산 대비 기여 매출 부족) | 빨강 |
| 캠페인 ROI 우수 | ROI ≥ 2.5 | 초록 (확장 후보) |
| 환불률 이상 | 직전 주 대비 +1.5%p 이상 | 노랑 |
| 자사앱 신규가입 급증 | 직전주 대비 +50% 이상 | 초록 (요인 추적) |

## 길이·톤 제약

- **A4 1~2장 안**. PDF 변환 시 표만 사용 (이미지 0개).
- 표는 마크다운 표. 셀 안에 줄바꿈 사용 금지.
- 임원이 5분 안에 핵심 파악 가능하게. 본문 산문 최소화, 표·알림 위주.
- 톤: 사실·숫자 중심. 추정은 "추정" 명시.
- 데이터에 없는 외부 추정 금지. 모르면 "데이터 없음" 명시.

## 산출물 형식

기본: 마크다운(.md). 본부장 검토 후 한 줄 요청으로 PDF용 HTML(monochrome 톤) 변환.

## 주의

- **김슬기 팀장 본인이 직접 회의 직전에 추가 질문을 던지는 흐름을 가정한다.** 보고서 1차 산출 후 "쿠팡이츠 매출 왜 떨어진 거야?" 같은 즉문에 동일 데이터 안에서 답할 수 있어야 한다 (외부 추정 금지).
- 캠페인 활성 기간이 보고 윈도 일부만 겹치는 경우, 겹친 일수만큼 예산을 일할 계산.
- 인스타 게시물의 캠페인 태그가 캠페인 마스터에 없는 값이면 "기타/일상"으로 분류.
