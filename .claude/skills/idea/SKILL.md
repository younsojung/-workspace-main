# idea - Idea Capture

대화 내용에서 아이디어/인사이트를 추출하여 `./40-personal/43-ideas/`에 저장.

## 워크플로우

1. **대화 분석** → 핵심 아이디어 추출
2. **제목 생성** (사용자가 힌트를 줬으면 반영, 아니면 자동 생성)
3. **저장 경로 결정**:
   - 기본: `./40-personal/43-ideas/YYYY-MM-DD_제목.md`
   - 이 아이디어가 **지식 위키 토픽급**이면 사용자에게 물어서 `./30-knowledge/00-wiki/`로 저장 제안
4. **문서 생성**
5. **결과 보고**

## 문서 구조

```markdown
---
date: YYYY-MM-DD
tags: [idea]
---

# [아이디어 제목]

## 맥락
[대화에서 이 아이디어가 나온 배경]

## 핵심 내용
[아이디어 본문 — 요약, 주장, 관찰]

## 실무 적용
[어떻게 활용할 수 있는지 — 구체적으로]

## 후속 행동
- [ ] [다음 스텝 1]
- [ ] [다음 스텝 2]

## Related
- (관련 노트, 토픽 — 있으면)
```

## 파일명 규칙

- `YYYY-MM-DD_주제-케밥케이스.md`
- 예: `2026-04-24_skills-vs-commands.md`
- 한글도 OK: `2026-04-24_위키-복리-구조.md`

## 00-wiki로 승격 판단

다음 조건이면 사용자에게 "00-wiki로 저장할까요?" 제안:
- 개념/원칙/프레임워크 성격 (일회성 아이디어 아님)
- 여러 대화에서 반복 등장할 것 같음
- 이미 wiki에 관련 페이지가 있음

## 원칙

- **맥락 반드시 포함** — "왜 이 아이디어가 나왔는지"가 가장 중요
- **실무 적용 중심** — 추상적으로 끝내지 말 것
- **후속 행동 1개 이상** — 쌓이기만 하고 실행 안 되면 의미 없음
- **태그 간소화** — 과도한 태그 금지 (`#idea` + 핵심 1~2개)
