# Web Application Design Principles

> Claude 내부 참조용. 사용자에게 직접 보여주지 않음.
> Phase 3 화면 설계 시 이 원칙을 적용하여 Screen/컴포넌트/플로우를 제안.

## 1. 화면 유형 분류

| 유형 | 목적 | 핵심 컴포넌트 | 사용 시점 |
|------|------|--------------|----------|
| List View | 엔티티 목록 조회 + 필터/검색 | DataTable, 검색바, 필터, 페이지네이션 | 관리할 항목이 여러 개일 때 |
| Detail View | 단일 엔티티 상세 + 관련 정보 | 상세 카드, 탭, 관련 목록, 액션 버튼 | 항목을 깊이 봐야 할 때 |
| Form View | 엔티티 생성/수정 입력 | 폼 필드, 유효성 메시지, 제출 버튼 | 데이터 입력/수정 시 |
| Dashboard View | 요약 지표 + 최근 활동 | KPI 카드, 미니 차트, 최근 목록 | 로그인 직후 랜딩 |
| Settings View | 설정/프로필 관리 | 폼, 토글, 섹션 구분 | 앱 설정 변경 시 |

## 2. 폼 설계 원칙

- **필수/선택 구분**: 필수 필드에 * 표시. 선택 필드는 "(선택)" 라벨
- **인라인 유효성 검증**: 포커스 아웃 시 즉시 피드백. 제출 시 재검증
- **에러 메시지**: 필드 바로 아래, 빨간 텍스트. "무엇이 잘못되었고, 어떻게 고쳐야 하는지"
- **멀티스텝 폼**: 3단계 이하. 진행률 표시(Stepper). 각 단계 독립 유효성 검증
- **자동저장**: 장문 입력 폼에서 30초마다 또는 포커스 아웃 시. "저장됨" 피드백
- **낙관적 업데이트**: 제출 후 즉시 UI 반영 → 백그라운드 서버 요청 → 실패 시 롤백
- **Destructive 액션**: 빨간 버튼 + 확인 다이얼로그. "정말 삭제하시겠습니까?" + 되돌릴 수 없음 명시

## 3. 테이블 설계 원칙

- **페이지네이션 vs 무한스크롤**: 관리 도구 = 페이지네이션 (정확한 위치 파악). 피드 = 무한스크롤
- **기본 정렬**: 최신순 (created_at DESC). 사용자 변경 가능
- **검색**: 테이블 상단. 주요 텍스트 필드 대상 full-text 검색
- **필터**: 상태, 날짜 범위, 카테고리. 활성 필터 수 Badge로 표시
- **벌크 액션**: 체크박스 선택 → 상단 액션 바 (삭제, 상태 변경 등)
- **인라인 편집**: 단순 필드(상태, 메모)에만. 복잡한 수정은 Detail View로
- **반응형 테이블**: 모바일에서 카드 뷰로 전환. 핵심 2-3컬럼만 표시
- **빈 테이블**: "아직 [엔티티]이(가) 없습니다" + CTA 버튼

## 4. 네비게이션 패턴

| 패턴 | 적합한 경우 | 비고 |
|------|------------|------|
| 사이드바 | 5개+ 메뉴, 관리 도구, SaaS | 접고 펼치기 가능. 아이콘 + 텍스트 |
| 상단 탭 | 2-5개 메뉴, 단순 앱 | 모바일에서 스크롤 탭 |
| 브레드크럼 | 3단계+ 깊이 | 항상 홈으로 돌아갈 수 있도록 |
| 모바일 하단 네비 | 모바일 중심 앱 | 3-5개 아이콘. 카카오톡 스타일 친숙 |

**규칙**:
- 현재 위치 항상 표시 (활성 메뉴 하이라이트)
- 뒤로가기로 이전 맥락 복귀 가능
- 3클릭 이내에 모든 핵심 기능 접근

## 5. 상태 배지 컨벤션

| 상태 | 색상 | HEX 계열 | 의미 |
|------|------|----------|------|
| 접수/대기 | Blue | #3B82F6 | 시작 전, 확인 대기 |
| 진행중 | Yellow/Amber | #F59E0B | 처리 중 |
| 완료 | Green | #22C55E | 성공적 종료 |
| 취소/거부 | Gray | #6B7280 | 비활성 종료 |
| 긴급/에러 | Red | #EF4444 | 즉시 조치 필요 |

**규칙**: Dot + 텍스트 조합. 색상만으로 의미 전달 금지 (색맹 대응).

## 6. Empty State 설계

**필수 구성**:
1. **일러스트 또는 아이콘**: 시각적 맥락 (무엇이 비어있는지)
2. **설명 텍스트**: "아직 [엔티티]이(가) 없습니다"
3. **CTA 버튼**: "첫 [엔티티] 만들기" → 생성 폼으로 이동

**안티패턴**: 빈 테이블만 보여주기, "No data" 한 줄, CTA 없는 빈 화면

## 7. 상태 관리 패턴

| 상태 유형 | 저장 위치 | 예시 |
|----------|----------|------|
| 서버 상태 | DB + React Query/SWR | 엔티티 데이터, 사용자 프로필 |
| URL 상태 | URL params/searchParams | 필터, 정렬, 페이지, 탭 |
| 클라이언트 상태 | Zustand/useState | 모달 열림, 사이드바 접힘 |
| 폼 상태 | React Hook Form | 입력값, 유효성 |

**규칙**: URL로 공유/북마크 가능해야 하는 상태는 반드시 URL에. 새로고침해도 유지.

## 8. 한국 비즈니스 앱 맥락

- **카카오톡 UX 친숙도**: 채팅형 인터페이스, 하단 시트, 알림 뱃지 패턴에 익숙
- **엑셀 대체 관점**: "엑셀보다 나은 이유"가 명확해야 채택됨. 자동 계산, 실시간 공유, 모바일 접근
- **보고서 겸용**: 화면 캡처해서 보고서에 붙임 → 캡처해도 읽히는 레이아웃
- **한글 라벨**: 영어 약어 대신 한글 (Status → 상태, Created → 생성일)
- **날짜 형식**: YYYY-MM-DD 또는 YYYY년 MM월 DD일
- **통화 형식**: ₩1,234,567 (천 단위 구분)

## 9. 안티패턴

| 안티패턴 | 문제 | 대안 |
|----------|------|------|
| 모달 속 모달 | UX 혼란, 뒤로가기 불가 | 별도 페이지 또는 사이드 시트 |
| 무한 네스팅 | 맥락 상실 | 브레드크럼 + 최대 3단계 |
| DB 구조 노출 | 사용자가 이해 불가 | 비즈니스 용어로 변환 |
| 과잉 설정 | 결정 피로 | 스마트 기본값 + 고급 설정 숨김 |
| 로딩 무표시 | 앱이 멈춘 것처럼 보임 | 스켈레톤 UI 또는 스피너 |
| 에러 무처리 | 사용자가 다음 행동을 모름 | 에러 메시지 + 재시도/돌아가기 |
| 확인 없는 삭제 | 실수로 데이터 손실 | 확인 다이얼로그 + Soft delete |
| 단일 화면에 모든 기능 | 인지 과부하 | 탭/단계로 분리 |
