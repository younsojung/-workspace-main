## /etimeline 특수 진단 (04 와 별도)

### URL 구조 이슈
- /ebook 과 /etimeline 이 **동일 콘텐츠** 반환 → SEO 중복 문제 가능
- 사용자 멘탈 모델에서 "윤소정의 생각 80편 = E-book = etimeline" 세 이름이 같은 것 → 정보 구조 단순화 여지

### "timeline" 이라는 키워드
- URL slug 가 **etimeline** = "ebook + timeline" 합성으로 추정
- 그러나 페이지 안에 *timeline 시각화* 없음 (No.79 → No.57 의 단순 역순 그리드일 뿐)
- **카피 기회**: "윤소정 14년 80편의 타임라인" 같은 시각화/스토리텔링이 들어가면 URL slug 와 콘텐츠가 일치

### 헤더 카피의 자산화
- "20대 / 30대 / 40대 윤소정" = **시간의 누적** 메시지
- 그러나 그리드 카드에는 "20대 작품 / 30대 작품" 같은 **시기 묶음 카테고리가 없음**
- 헤더가 약속한 *시간 변화 경험*을 그리드가 이행하지 못함 = **카탈로그 IA 구멍**

### 가격 의사결정 보조 부재
- 단권 13,000원 × 80편 = 약 1,040,000원
- 풀세트 780,000원 = 약 260,000원 절약 (25%)
- 그러나 **이 절약 카피가 어디에도 없음** — 풀세트 카드 옆에 "단권 80개 ≒ 1,040,000원, 지금 ₩260,000 절약" 한 줄이면 풀세트 전환 ↑

### 추천 진입점 부재
- 80편 중 *"이 책으로 시작하세요"* 큐레이션 없음
- HOME 리뷰에서 가장 많이 언급된 책 = 깨끗한 학습력(No.63) → "입문 추천" 라벨이 카드에 있으면 의사결정 ↓
