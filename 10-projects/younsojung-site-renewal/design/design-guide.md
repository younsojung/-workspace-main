# 디자인 가이드 — 폰트 · 여백 · 줄간격

> Claude와 사이트/콘텐츠 작업할 때 읽는 레퍼런스. 세 사이트의 **실제 production CSS**에서 토큰을 직접 추출해 정리(2026-05-24). 추측 아님.

---

## 0. 세 레퍼런스가 똑같이 지키는 5원칙

성격이 다 다른데도(잡지 / 커머스 / 셀렉트샵) 공통으로 합의하는 규칙. 이게 핵심.

1. **폰트는 1~2종으로 끝낸다.** 한글은 셋 다 Pretendard 아니면 Noto. 멋은 "세리프를 영문 제목·디스플레이에만" 떨구는 식으로 절제해서 냄(POV).
2. **위계를 굵기로 만들지 않는다.** 셋 다 제목조차 weight 400~500. 700은 거의 안 씀. 위계는 **크기 + 여백 + 폰트 전환**으로.
3. **본문 자간은 손대지 않는다(0 / normal).** 좋은 한글 폰트는 자간이 이미 설계돼 있음. *단, 큰 디스플레이 헤딩의 음수 트래킹과 작은 라벨의 넓은 트래킹은 의도적 예외*(아래 참조).
4. **줄간격은 "제목 좁게, 본문 넓게."** 큰 글씨일수록 행간을 줄이고(1.0~1.36), 읽는 본문은 넉넉히(1.4~1.7). 글자가 클수록 행간이 시각적으로 벌어 보이기 때문.
5. **여백을 아끼지 않는다.** 섹션 사이 80px+, 콘텐츠 중앙 정렬. 정보를 빽빽이 채우지 않음.

### 세 사이트 수치 비교

| | Magazine B | 29CM | Point of View |
|---|---|---|---|
| 한글 폰트 | Noto Sans KR | **Pretendard 단일** | Pretendard |
| 영문/제목 | Beausite(자체) | Pretendard | **Larish 세리프**(영문 제목만) |

## 1. 폰트 (Typography)

이 프로젝트는 `identity-v1.html`에서 이미 방향을 박았음 — **POV식 "에디토리얼 세리프"**:

```css
--serif: 'Noto Serif KR', serif;     /* 본문 + 제목 (에디토리얼 톤) */
--sans:  'Inter', system-ui, sans-serif;  /* 영문 라벨 · 태그 · 숫자 */
```

**역할 분담**
- **세리프(Noto Serif KR)** = 한글 본문·제목·인용. 진지하고 사색적인 "생각의 페이스메이커" 톤.
- **산세리프(Inter)** = 영문 라벨(`LEARN.` `SHARE.`), 태그, 타임라인 숫자, 영문 캡션. 구조·메타 정보용.

> **읽기 페이지는 예외 검토**: 위 세리프 본문은 *identity/about 같은 짧은 에디토리얼*에 어울림. 생각구독 본문·ebook 같은 **장문 읽기 페이지**는 세리프가 눈을 피로하게 할 수 있음 → 그 페이지는 본문을 **Pretendard 산세리프 16~17px / 행간 1.7**로 가는 것을 권장(29CM·POV 본문 패턴). 제목만 Noto Serif KR 유지해서 대비.

---

## 2. 폰트 크기 (Type Scale)

본문 base **16px**. 좁은 스케일(약 1.125~1.25배)로 차분하게. 세 레퍼런스 모두 헤딩을 과하게 키우지 않음.

| 토큰 | 크기 | 용도 |
|---|---|---|
| `--text-caption` | 12~14px | 태그, 메타, 영문 캡션 |
| `--text-body` | **16px** | 기본 본문 |
| `--text-body-lg` | 18px | 강조 본문, 소제목 |
| `--h3` | 20~24px | 소제목 |
| `--h2` | 24~32px | 섹션 제목 |
| `--h1` | clamp(2rem, 6vw, 5.5rem) | 디스플레이 히어로(반응형) |

> 읽기 페이지 본문은 16 → **17px**로 한 단계 키우기.

## 3. 줄간격 (Line-height) — "제목 좁게, 본문 넓게"

```css
--lh-display:  1.05;   /* 히어로 대형 영문 (identity-v1: 1.05) */
--lh-heading:  1.2~1.3;
--lh-body:     1.6;    /* identity-v1 본문 기본 */
--lh-reading:  1.7;    /* 장문 읽기 페이지 (한글) */
--lh-quote:    1.5;    /* 인용문 */
```

- 디스플레이/큰 제목일수록 **1.0~1.3**으로 조이기.
- 한글 읽는 본문은 **1.6~1.7**로 풀기 (한글은 영문보다 행간을 더 줘야 편안함).
- 29CM 규칙: "큰 글씨는 좁게(1.36), 본문은 넓게(1.4)." 한글이라 이 가이드는 본문을 더 키운 1.6~1.7로 조정.

---

## 4. 자간 (Letter-spacing)

**기본 원칙: 본문은 0 / normal — 손대지 않는다.** 단 에디토리얼 효과를 위한 두 가지 *의도적* 예외만 허용(identity-v1 적용 패턴):

| 상황 | 자간 | 이유 |
|---|---|---|
| 한글/영문 본문 | `0` (normal) | 폰트 기본 신뢰 |
| 큰 디스플레이 헤딩 | `-0.02 ~ -0.04em` | 큰 글자는 자간이 벌어 보임 → 살짝 조임 |
| 작은 영문 라벨·태그 | `0.2 ~ 0.45em` | 넓은 트래킹으로 "라벨" 질감 (대문자와 함께) |

→ 이 두 예외 **외에는 자간을 건드리지 말 것.** 한글 본문에 음수 자간 깔지 않기.

## 5. 여백 (Spacing) — 8의 배수 리듬, 과감하게

```css
/* 간격 스케일 (8pt 리듬) */
8px · 16px · 24px · 32px · 48px · 80px

--section-gap:  80px;     /* 섹션 사이 (세 레퍼런스 공통) */
--content-max:  1200~1380px;  /* 페이지 전체 폭, 중앙 정렬 */
--reading-max:  640~720px;    /* ★ 읽는 본문 한 줄 폭 */
--pad-x:        8vw;      /* 좌우 여백 (identity-v1: 8vw) */
```

**제일 중요한 한 줄**: 읽는 본문은 페이지 폭이 아니라 **한 줄 640~720px(약 60ch)로 좁혀라.** 한 줄이 너무 길면 눈이 다음 줄을 못 찾음. (identity-v1도 인용문에 `max-width: 60ch` 사용 중 — 이미 맞는 방향.)

- 섹션 사이는 과감하게 **80px+** (Magazine B는 100~128px까지).
- 요소 사이는 좁게(8~24px), 섹션 사이는 크게 → **리듬 대비**가 호흡을 만듦.
- 한글은 `word-break: keep-all;` **필수** (단어 단위 줄바꿈, Magazine B 디테일).

## 6. 색 (Color)

identity-v1은 **순(純) 모노크롬**:

```css
--black: #0a0a0a;  --white: #fafafa;
--gray-100: #f5f5f5;  --gray-300: #d4d4d4;
--gray-500: #737373;  --gray-700: #404040;
```

> **선택지**: POV처럼 순백 대신 **웜 크림(#FAF6ED 계열)** 바탕을 쓰면 종이·인쇄물 같은 따뜻한 에디토리얼 톤이 됨. "생각구독 = 매달 받아보는 글" 정체성과 잘 맞을 수 있음. → 사용자 결정 필요(Open).

## 7. Do / Don't 요약

**Do**
- 폰트 2종(Noto Serif KR + Inter)으로 제한, 위계는 크기·여백으로.
- 한글 본문 행간 1.6~1.7, 제목 1.0~1.3.
- 읽는 본문 한 줄 폭 640~720px로 좁히기.
- 섹션 사이 80px+, `word-break: keep-all`.
- 자간은 본문 0, 디스플레이/라벨만 의도적 예외.

**Don't**
- 제목을 굵게(700+)만 키워서 위계 만들기 ❌ (촌스러움의 신호)
- 한글 본문에 음수 자간 깔기 ❌
- 본문 한 줄을 화면 끝까지 늘리기 ❌
- 폰트 3종 이상 섞기 ❌
