# 디자인 가이드 — 폰트 · 여백 · 줄간격

> Claude와 사이트/콘텐츠 작업할 때 읽는 레퍼런스. 세 사이트의 **실제 production CSS**에서 토큰을 직접 추출(2026-05-24). 추측 아님.
> **운영 원칙은 비넬리 타이포** — 카드뉴스와 동일한 기준. 큰 글씨로 외치지 않는다.
> 💾 **실행 버전**: [`trus-design.css`](trus-design.css) — 이 문서가 *소스*, CSS가 *구현*. 페이지에서 `<link>` 한 줄로 토큰 전체 적용.

---

## ★ 비넬리 원칙 (이 프로젝트의 기준)

> "You can say everything you want with very few type sizes." — Massimo Vignelli

1. **크기는 소수만 쓴다.** 작은 / 본문 / 중간 / 큰, 4단계면 충분. 더 만들지 않는다.
2. **크기 대비를 절제한다.** 단계 사이는 **일정한 비율(1.25배)**로 연결. 큰 글씨가 본문의 5배씩 튀지 않는다 (가장 큰 글씨도 본문의 **2배 이내**).
3. **강조는 크기가 아니라 굵기·여백·위치로.** 뭔가 도드라지게 하고 싶으면 글씨를 키우지 말고 → 굵게(weight) 하거나, 둘레 여백을 비우거나, 위치를 바꾼다.

> ⚠️ v1에서 디스플레이를 88px(`clamp(2rem,6vw,5.5rem)`)까지 키운 건 비넬리 원칙 위반이었음. v2에서 비율 재정렬. `identity-v1.html`의 히어로(최대 6.5rem)도 같은 이유로 축소 검토 대상.

## 0. 세 레퍼런스가 똑같이 지키는 5원칙

성격이 다 다른데도(잡지 / 커머스 / 셀렉트샵) 공통으로 합의하는 규칙.

1. **폰트는 1~2종으로 끝낸다.** 한글은 셋 다 Pretendard 아니면 Noto.
2. **위계를 크기 대비로만 만들지 않는다.** 셋 다 제목조차 weight 400~500, 크기도 과하게 안 키움. 위계는 **여백 + 위치 + (필요 시) 굵기**로.
3. **본문 자간은 손대지 않는다(0 / normal).** *단, 큰 디스플레이의 음수 트래킹과 작은 라벨의 넓은 트래킹은 의도적 예외*.
4. **줄간격은 "제목 좁게, 본문 넓게."** 큰 글씨일수록 행간 1.0~1.36, 읽는 본문은 1.6~1.7.
5. **여백을 아끼지 않는다.** 섹션 사이 80px+, 콘텐츠 중앙 정렬.

### 세 사이트 수치 비교

| | Magazine B | 29CM | Point of View |

| 배경 | 순백 | 흰색 | 웜크림 #FAF6ED |

## 1. 폰트 (Typography) — 기능으로 나눈다

```css
--display: 'Pretendard Variable', Pretendard, system-ui, sans-serif;  /* 제목·라벨·짧은 선언 */
--serif:   'Noto Serif KR', serif;   /* 여러 줄로 흐르는 본문 단락 */
```

**규칙: 크기가 아니라 "기능"으로 폰트를 나눈다.**

| 폰트 | 적용 대상 |
|---|---|
| **Pretendard** | 제목 · 라벨 · 태그 · 목록 · **짧은 선언/정의 문장** (예: "기획 = 선생님이 수업 전에 하는 것", "기획하는 교육자", 영문 라벨) |
| **명조(Noto Serif KR)** | **본문처럼 여러 줄로 흐르는 단락** (읽는 글) |

- 짧은 한 줄짜리는 선언·정의·인용이라도 **Pretendard**. 명조는 *진짜 읽는 본문 단락*에만.
- **영문 디스플레이 세리프(POV의 Larish 같은 것)는 "그때그때 선택"** — 특정 페이지의 영문 제목에 쓰고 싶을 때만.
- 굵기 3단으로 단순화: **Regular 400(본문) · Medium 500(라벨·강조) · Bold 700(제목·핵심).**

> ※ 장문 읽기 페이지(생각구독·ebook)는 본문이 길어지므로 명조 비중이 커짐. 짧은 랜딩·정체성 페이지는 거의 Pretendard, 본문 단락만 명조.

## 1-B. ★ 호흡 줄바꿈 — 윤소정식 폰트 디자인의 생명

> **"잘 읽히게 만드는 것"이 윤소정 타이포의 핵심.** 잘 읽힌다 = 소리 내어 읽었을 때 자연스럽다.

- **소리 내어 읽어본다.** 숨을 쉬는 자리가 있다.
- 그 **숨 쉬는 타이밍에 아래로 내려쓴다(`<br>` 또는 줄바꿈).** 그래야 글씨가 눈에 잘 들어온다.
- 띄어쓰기(공백)와 내려쓰기(줄바꿈)를 의도적으로 설계 — 기계적 줄바꿈(폭에 맞춰 아무데서나 끊김)에 맡기지 않는다.

**예시**
```
❌  "선생님이 수업 전에       (숨을 못 쉬는 자리에서 끊김)
     하는 것."
✅  "선생님이                 (주어 뒤 = 숨 쉬는 자리)
     수업 전에 하는 것."
```

- 한 문장도 호흡 단위로 내려쓰면 리듬이 생긴다 ("나는 내가 딱 해본 것만큼만 / 공유합니다").
- **적용 범위 = 짧은 핵심 문장·리드·제목·인용.** 임팩트를 줘야 하는 한두 줄에만.
- ⚠️ **길게 설명하는 "그냥 본문" 단락은 붙여서 흐르게 둔다.** 억지 호흡 줄바꿈 ❌ — `keep-all`로 단어만 보호하고 자연스럽게 흐르게. (본문까지 끊으면 답답하고 완성도 떨어짐.)
- 모든 비주얼(사이트·카드뉴스·핸드아웃)에 동일 적용. → [[비넬리 타이포 원칙]]과 한 쌍.

## 2. 폰트 크기 (Type Scale) — 비율 1.25로 재정렬

**큰/중간/작은을 일정한 비율(1.25배, Major Third)로 묶음.** 가장 큰 글씨도 본문의 2배를 넘기지 않음.

| 단계 | px | rem | 용도 |
|---|---|---|---|
| 작은 글씨 (S) | **13** | 0.8 | 캡션 · 메타 · 태그 |
| 보조 | 14 | 0.875 | 부가 설명 |
| **본문 (읽기)** | **15** | 0.9375 | 읽는 본문 ✅ 확정 |
| 중간 글씨 (M) | **20** | 1.25 | 소제목 · 리드문 |
| 큰 글씨 (L) | **25** | 1.563 | 섹션 제목 |
| 디스플레이 (XL) | **31** | 1.953 | 페이지 히어로 (상한) |

> ※ **본문 = 15px 확정** (16px보다 차분·에디토리얼, 작은 글씨와의 대비가 부드러움). 나머지 스케일(13·14·20·25·31)은 유지.

```css
--text-s:    0.8rem;     /* 13px */
--text-body: 0.9375rem;  /* 15px — 읽기 본문 */
--text-m:    1.25rem;    /* 20px */
--text-l:    1.563rem;  /* 25px */
--text-xl:   1.953rem;  /* 31px — 최대. 이보다 키우지 않는다 */
```

- **3-tier 멘탈 모델**: 작은(13~14) · 중간(20) · 큰(25). 본문 15는 작은 글씨 가까운 기준점. → 강조는 이 안에서 한 단계 올리고 **굵게**.
- 반응형이 필요하면 `clamp()`를 쓰되 **상한을 ~31px(2rem)로 잡는다.** (`clamp(1.5rem, 4vw, 1.953rem)` 식)
- 더 강한 임팩트가 필요한 단 하나의 표지급 히어로에 한해 예외적으로 더 키울 수 있으나, **그건 비넬리 예외**임을 인지하고 페이지당 1회로 제한.

## 3. 줄간격 (Line-height) — "제목 좁게, 본문 넓게"

```css
--lh-display: 1.1;    /* 큰·디스플레이 (25~31px) */
--lh-heading: 1.3;    /* 중간 제목 (20px) */
--lh-body:    1.7;    /* 한글 본문 — 넉넉히 */
--lh-quote:   1.5;    /* 인용문 */
```

- 글씨가 클수록 행간을 조인다(1.1~1.3). 한글 읽는 본문은 1.6~1.7로 푼다(한글은 영문보다 행간을 더 줘야 편안).

## 4. 자간 (Letter-spacing)

**기본: 본문 0 / normal — 손대지 않는다.** 의도적 예외 2가지만:

| 상황 | 자간 |
|---|---|
| 한글/영문 본문 | `0` (normal) |
| 큰 디스플레이(L·XL) | `-0.02 ~ -0.03em` (큰 글자만 살짝 조임) |
| 작은 영문 라벨·태그 | `0.2em` 내외 (대문자와 함께) |

→ 한글 본문에 음수 자간 ❌.

## 5. 여백 (Spacing) — 8의 배수 리듬, 과감하게

```css
8px · 16px · 24px · 32px · 48px · 80px
--section-gap: 80px;        /* 섹션 사이 (세 레퍼런스 공통) */
--content-max: 1200~1380px; /* 페이지 폭, 중앙 정렬 */
--reading-max: 640~720px;   /* ★ 읽는 본문 한 줄 폭 (약 60ch) */
--pad-x:       8vw;
```

**제일 중요한 한 줄**: 읽는 본문은 페이지 폭이 아니라 **한 줄 640~720px(60ch)로 좁힌다.** 한 줄이 길면 눈이 다음 줄을 못 찾음.

- 요소 사이 좁게(8~24px), 섹션 사이 크게(80px+) → 리듬 대비가 호흡을 만든다.
- 한글은 `word-break: keep-all;` **필수**.
- 강조하고 싶을 때: 글씨 키우지 말고 **둘레 여백을 더 비운다.** (비넬리식)

## 6. 색 (Color) — TRUS 브랜드 팔레트

공식 TRUS 3색 기준 (사용자 제공 무드보드, 2026-05-24). 브랜드 서사 **틀에박힌(BOUND) → 깨어나는(AWAKENING) → 확장되는(EXPANDING)**.

```css
--trus-black:   #121212;   /* TRUS BLACK — 텍스트·잉크·1차 버튼 */
--trus-white:   #FFFFFF;   /* TRUE WHITE — 기본 배경 */
--educate-pink: #D84D8D;   /* TRUE EDUCATE PINK — 시그니처 액센트 (로즈핑크) */
--ink: var(--trus-black);  /* 컴포넌트(§8)에서 쓰는 별칭 */
/* 보조 그레이 */
--gray-100:#f5f5f5; --gray-300:#d4d4d4; --gray-500:#737373; --gray-700:#404040;
```

- **기본 구성**: TRUE WHITE 배경 + TRUS BLACK 텍스트. (POV식 웜크림은 TRUS 톤이 아니므로 비표준 — 특수 페이지에서만.)
- **TRUE EDUCATE PINK = 시그니처.** *각성·변화*를 상징. 강렬하므로 **더더욱 절제** — 화면의 5% 미만.

## 6-B. EDUCATE PINK 사용 규칙 — ✅ 액센트 전용 (절제)

**핑크는 절제해서 쓴다.** 1차 텍스트·버튼은 BLACK, 핑크는 *핵심 강조 1개·활성(active) 상태·링크·작은 하이라이트*에만. **화면의 5% 미만.** (29CM·Magazine B식 — 컬러는 점처럼.)

- ✅ 한 페이지에 핑크는 "여기 딱 하나" 싶은 자리에만 (예: 구분선 하나, 핵심 단어 하나).
- ❌ 넓은 면적·배경·본문 다수·버튼 여러 개를 핑크로 ❌.
- 데모(`identity-v1.html`): 히어로 구분선 + `끝・까・지` 단어, 딱 2군데.

## 8. 컴포넌트 (Components) — 한 번 정하면 재사용

**그림자 ❌ → hairline 보더로.** 이 브랜드는 입체감(그림자)이 아니라 1px 선과 여백으로 구조를 만든다(너의 시그니처).

```css
--radius: 2px;   /* 거의 직각 — 에디토리얼. 더 날카롭게 하려면 0 */

/* 버튼 — hover 시 채움↔외곽선 반전 */
.btn { font-family:var(--display); font-weight:600; font-size:1rem;
  padding:14px 28px; border-radius:var(--radius); letter-spacing:-0.01em;
  display:inline-block; transition:all .2s ease; cursor:pointer;
  min-height:44px; }            /* 모바일 탭 최소 44px */
.btn-primary   { background:var(--ink); color:#fff; border:1px solid var(--ink); }
.btn-primary:hover   { background:transparent; color:var(--ink); }
.btn-secondary { background:transparent; color:var(--ink); border:1px solid var(--ink); }
.btn-secondary:hover { background:var(--ink); color:#fff; }

/* 링크 — 본문 내 밑줄, 살짝 띄움 */
a { color:inherit; text-underline-offset:3px; text-decoration-thickness:1px; }
a:hover { opacity:.6; }

/* 구분선 (hairline) — 시그니처 */
.rule        { height:1px; background:var(--gray-300); border:0; margin:0; }
.rule-strong { background:var(--ink); }
.rule-short  { width:60px; height:1px; background:var(--ink); }  /* 히어로 디바이더 */

/* 태그·뱃지 */
.tag   { font-family:var(--display); font-size:.8rem; font-weight:600;
         letter-spacing:.1em; color:var(--gray-500); }
.badge { display:inline-block; border:1px solid var(--gray-300);
         padding:4px 12px; border-radius:999px; font-size:.8rem; }

/* 카드 — 보더만, 그림자 없음 */
.card  { border:1px solid var(--gray-300); padding:28px 24px; }

/* 인용 블록 (pull-quote) — 위아래 짧은 선으로 감쌈 */
.pullquote { text-align:center; max-width:60ch; margin:0 auto; font-size:1.25rem;
             font-weight:500; line-height:1.5; }
.pullquote::before, .pullquote::after {
  content:''; display:block; width:40px; height:1px; background:var(--ink); margin:1rem auto; }
```

## 9. 반응형 · 모바일 — 유입의 대부분이 모바일

브레이크포인트(레퍼런스 정렬): **480px(모바일) · 768px(태블릿) · 1024px · 1280px**.

```css
@media (max-width: 768px){
  :root { --pad-x: 6vw; --section-gap: 48px; }  /* 여백 축소 */
}
@media (max-width: 480px){
  :root { --pad-x: 20px; --section-gap: 40px; }
}
```

- **본문은 16px 밑으로 내리지 않는다.** (iOS에서 16px 미만 입력 시 자동 줌·가독성 저하.) 작은 글씨도 14px가 하한.
- 디스플레이는 `clamp()`로 자동 축소 — 모바일에서 상한이 아니라 하한(`1.4rem`)에 가까워짐.
- 읽는 본문 폭은 모바일에서 화면폭 − 좌우 패딩(자동). `--reading-max`는 데스크탑 한정.

### ★ 호흡 줄바꿈의 모바일 처리 (중요)

데스크탑에서 손으로 넣은 `<br>`가 모바일 좁은 폭에선 엉뚱한 데서 끊긴다. **데스크탑 줄바꿈은 모바일에서 끄고 자연 흐름에 맡긴다:**

```css
br.pc { display: inline; }
@media (max-width: 600px){ br.pc { display: none; } }   /* 모바일선 자연 줄바꿈 */
br.mo { display: none; }
@media (max-width: 600px){ br.mo { display: inline; } }  /* 모바일 전용 줄바꿈 */
```
```html
<p>자퇴생에서 파워블로거 1세대로,<br class="pc">선생에서, 기획자로…</p>
```
- 긴 본문 단락의 `<br>`는 전부 `class="pc"` — 모바일에선 `keep-all`로 단어 단위 자연 줄바꿈.
- 짧은 카피·제목은 모바일에서도 호흡 줄바꿈이 살아있게 두거나, `br.mo`로 모바일 전용 호흡점을 따로 지정.

## 10. 이미지 · 한글 조판 디테일

### 이미지 · 사진
- **비율 고정**: 카드/인스타 4:5(세로), 와이드 3:2 또는 16:9. `object-fit: cover`로 채우고 늘리지 않음.
- **톤 일관성**: 모노 브랜드와 충돌 안 나게 — 사진은 풀컬러로 쓰되 과채도 ❌, 또는 흑백/저채도로 통일하면 더 에디토리얼.
- **캡션**: 사진 아래 `--text-s`(13px) Pretendard, `--gray-500`. 좌측 정렬.
- **그림자 없이** 보더 또는 풀블리드.

### 한글 조판 디테일
- **`word-break: keep-all;`** — body 기본. 제목·라벨·짧은 문장·**왼쪽정렬 본문**에서 단어가 음절 중간에 깨지는 것(`키워가/는`) 방지.
- **예외 — 양쪽정렬 본문은 `word-break: normal`**: 양 끝을 빽빽이 채우려면 자연 줄바꿈(어절 중간 허용)이 필요. 양쪽정렬이라 어절이 나뉘어도 가지런해 보임(책·신문식).
- ★ **고아(orphan) 금지** *(왼쪽정렬/ragged 본문에서 특히)*: 조사(는/을/를/이/가)·단일 음절·문장 끝 글자가 줄 끝에 혼자 떨어지면 안 된다. 안전망 `text-wrap: pretty;`. **그래도 남으면 — 줄바꿈 위치를 옮기거나, 필요하면 전체 폰트 크기를 미세 조정해서라도 완성도를 지킨다.** (양쪽정렬 블록에선 어절이 나뉘어도 자연스러움.)
- **한 호흡 = 한 줄**: 한 번에 쭉 읽히는 구절은 쪼개지 말고 붙여 둔다. 줄바꿈(`<br>`)은 숨 쉬는 자리에서만. 데스크탑 `<br>`는 모바일에서 끈다(`br.pc`, §9).
- **정렬**: 붙여 쓰는 본문 단락 = **양쪽정렬(`text-align: justify` + `word-break: normal`)** → 어절이 줄 끝에서 나뉘어도 빽빽이 꽉 찬 가지런한 블록(책·신문식). 호흡 줄바꿈한 짧은 문장·리드 = **왼쪽정렬 + `keep-all`**(단어 보호, 짧은 줄 안 벌어지게).
- **영문·숫자 혼용**: Pretendard가 라틴·숫자까지 커버 → 본문이 명조여도 숫자/영문은 자연스럽게 섞임. 표·수치는 `font-variant-numeric: tabular-nums;`로 자릿수 정렬.
- **본문 내 강조**: 색 ❌ → **굵게(600~700)**. 특별한 한 단어는 자간 벌리기(`끝・까・지`)처럼 *간격*으로. 밑줄은 링크에만.
- **따옴표**: 곧은 따옴표(`"`) 말고 둥근 따옴표(`" "` `' '`) 사용. 문장부호 일관.
- 한 줄이 마침표·쉼표로 시작하지 않게 (keep-all이 대부분 처리).

## 11. Do / Don't 요약

**Do**
- 폰트 Pretendard 기본, 위계는 굵기(400/500/700)·여백으로.
- 크기는 4단계(13·16·20·25, 비율 1.25)만. 디스플레이 상한 31px.
- 한글 본문 행간 1.6~1.7, 제목 1.1~1.3.
- 읽는 본문 한 줄 폭 640~720px.
- 강조 = 굵게 + 여백 (크기 ❌).
- 섹션 사이 80px+, `word-break: keep-all`.
- 줄바꿈은 소리 내어 읽어 호흡점에 직접 `<br>`. 데스크탑 `<br>`는 모바일에서 끈다(`br.pc`).
- 버튼·링크는 잉크 블랙, hover 시 채움↔외곽선 반전.
- 카드·박스는 그림자 ❌ → hairline 보더로.
- 모바일 본문 16px 유지(밑으로 ❌).

**Don't**
- 강조한다고 글씨를 크게 키우기 ❌ (비넬리 위반 — v1의 실수)
- 디스플레이를 본문의 2배 넘게 키우기 ❌
- 폰트 3종 이상 섞기 ❌
- 한글 본문에 음수 자간 ❌
- 본문 한 줄을 화면 끝까지 늘리기 ❌
- 화려한 컬러로 강조 ❌ (액센트는 5% 미만)
- box-shadow로 입체감 ❌
