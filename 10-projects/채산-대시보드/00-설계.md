# 채산 대시보드 — 설계 문서

> **목표**: 매월 통장·카드 거래 데이터를 자동으로 분류 → 채산표 산출 → 대시보드 표시
> **운영 원칙**: 분류 규칙은 코드가 아닌 **YAML 설정 파일**에 둠 → 새 거래처/패턴이 생기면 YAML만 수정
> **반복 단위**: 월 1회 데이터 갱신 → 대시보드 자동 업데이트

---

## 🏗 전체 구조

```
[원본 데이터]              [설정]               [엔진]                [출력]
통장 CSV     ─┐                          ┌─→ 거래단위 분류 CSV
               │   ┌─ taxonomy.yaml ─┐  │
카드 XLS     ─┼─→ ┤ revenue.yaml    ├─→ │─→ 월별 채산표 CSV  ─→ [대시보드]
               │   └─ exclusions.yaml┘  │
미분류 룰   ─┘                          └─→ unclassified.csv
                                              (수동 검토 → YAML 추가)
```

### 폴더 구조 (확정안)
```
10-projects/채산-대시보드/
├── 00-설계.md                       # 이 문서
├── config/
│   ├── taxonomy.yaml               # 지출 분류 체계 + 자동 분류 규칙
│   ├── revenue.yaml                # 매출 분류 (POS·구독·강의료)
│   └── exclusions.yaml             # 영업외/제외 항목
├── data/
│   ├── raw/                        # 원본 (월별 파일 누적)
│   │   ├── bank_YYYYMM.csv
│   │   └── card_YYYYMM.xls
│   ├── processed/
│   │   ├── transactions.csv        # 분류된 거래 단위 (누적)
│   │   └── monthly_pnl.csv         # 월별 채산표
│   └── unclassified.csv            # 자동분류 실패 (검토용)
├── src/
│   ├── ingest.py                   # 신규 raw 파일 → 분류 → processed 갱신
│   ├── classify.py                 # YAML 룰 기반 분류 엔진
│   ├── aggregate.py                # 월별·카테고리별 집계
│   └── dashboard.py                # 대시보드 (Streamlit 예정)
└── outputs/
    └── (그래프·리포트)
```
