# 생각구독 발행 워크플로우 — 81호 기준 8단계

> 81호(슈퍼샘플의 시대) 작업하면서 정립된 순서. 다음 호부터 봇이 이 8단계 그대로 진행.

## 8단계

### 1. 한 달 정리 — Raw Material 인터뷰
사람 / 사건 / 성과 3축으로 10가지 펼침.

**인터뷰 4질문 (각 사건마다)**:
- 의미있던 순간은 언제였어?
- 그래서 배운 게 뭐야?
- 행동으로 성과를 낸 건 뭐야?
- 다음에 학습을 더 하고 싶은 건 뭐야?

저장: `concept.md`에 raw 10건

### 2. 핵심 개념 추출
10건의 공통점 → 한 단어. 슈퍼샘플 / 페이스메이커 같은 시대 단어.

### 3. 컨셉 + 부제 + 키 퀘스천
- **컨셉**: "X월. ~~의 시대" 형태 한 줄
- **부제**: *카피라이터 톤*. 통념 역설·신체 동사·21자 단호. ("배우는 시대는 끝났다. 이제는 베끼는 시대다.")
- **키 퀘스천**: 독자가 끝까지 묻게 하는 한 문장 ("나는 베낄 만한 샘플인가?")

부제 룰:
- AI·강사·30대·18년차 같은 *타깃 한정 어휘* 빼기
- 포모(공포 후킹) 빼기 — 욕망·발견·통찰 기반
- "이 글 사고 싶다"를 만드는 한 줄
- 한 번 들었을 때 머릿속에 *그림*이 그려져야 함

### 4. 표지 무드보드 + 디자이너 브리프
핀터레스트 원본 + AI 시안 + 카피 위치·격자 구조 + 80호 시리즈와의 연속성.
저장: `cover_brief.md`

### 5. 이야기 설계 (Story Design)
- 큰 아크 박기 (감정 곡선 / 시간 흐름 / 대비)
- 첫 챕터·마지막 챕터 후보 박기
- 80편 코퍼스에서 비슷한 톤 호 1-2개 참조 (`grep -l "키워드" _cards/`)

### 6. 목차 (TOC)
INTRO + 챕터 3~5개 + CLOSING.
**HTML로 같이 체크** (`toc.html` — 목차 + 본문 미리보기 + 챕터별 복사 버튼).
**글 쓰면서 목차 수정 OK** — *말이 되는 목차*가 최우선.

### 7. 본문 작성 (챕터별 사이클)
한 챕터씩 — 다음 순서:
1. 사용자 raw 받음 → `drafts.md`에 원본 보존
2. 봇 정리체 변환 → 채팅창에 초안 보여줌
3. `toc.html`에 본문 미리보기로 즉시 반영
4. 사용자 수정 피드백 → 갱신
5. OK → `body_NN.md` 저장 (정리체)
6. 다음 챕터 진입

각 단락 = `body-block` 단위 + 📋 복사 버튼 (구글 문서 이전용)

### 8. 전체 옮기기 (구글 문서 이전)
모든 챕터 통합 → `full_manuscript.md` + `full-manuscript-export.html` (전체 복사 버튼).
사용자가 구글 문서에 Cmd+V → 거기서 최종 편집·발행.

## 본문체 규칙 (81호에서 박힘)

- **INTRO·CLOSING**: 편지체 ("~어", "~지", "안녕? 나의 영혼의 친구" 톤)
- **챕터 01~04**: **정리체** ("~다." 스스로 정리하는 어투)

체 변환 예시:
- "누군가 묻더라" → "누군가 묻는다"
- "답이 바뀌었어" → "답이 바뀌었다"
- "근데 본질은 하나야" → "그러나 본질은 하나다"

## 81호에서 박힌 추가 원칙

| 원칙 | 박힌 내용 |
|---|---|
| 카피 원칙 | 직관·실용·누구나. 시적/추상 ❌. 감성 = 친밀한 직설 |
| 정의 합의 | 본질·정체성 단어는 *사용자가 박음*. AI는 후보 펼치고 대기 |
| 외부 공개 숫자 가드 | 활성 구독자·누적 매출·재구독률 비공개. 카피 X |
| HTML로 같이 체크 | `toc.html` 한 페이지에서 목차+본문 실시간 확인 |
| drafts.md 보존 | 사용자 raw 원본 누적. 다음 호 봇이 참조 |
| 80편 코퍼스 활용 | 비슷한 톤 호 검색 + 4축 인용 추출 (Commit/Issue/Code/Fork) |
| 팩트체크 | 외부 자료로 정의·실무 검증 후 반영 |
| 단락별 복사 | 구글 문서 이전용 복사 버튼 (HTML + Plain) |
