# Wiki Schema

> "The knowledge is compiled once and then kept current, not re-derived on every query." — Karpathy
> 이 파일은 AI와 함께 co-evolve 한다. 위키 운영 중 발견된 패턴은 여기에 반영.

## 구조

```
00-wiki/
├── SCHEMA.md         # 이 파일 (CLAUDE.md에서 @import, AI와 co-evolve)
├── index.md          # 전체 토픽 카탈로그 (카테고리별, 메타데이터 포함)
├── log.md            # 시간순 활동 기록 (append-only, parseable)
├── README.md         # 운영 가이드
└── [토픽].md          # 토픽 페이지 (flat, 케밥케이스 한글 또는 영어)
```

## 3계층

| 계층 | 위치 | 역할 | 누가 관리 |
|------|------|------|----------|
| Raw Sources | 10-projects/, 41-daily/, 00-inbox/ 등 | 원본 소스. **불변** — 읽기만, 수정 안 함 | 사용자 |
| Wiki | **00-wiki/** | 소스에서 추출한 지식이 통합되어 복리로 축적 | AI가 쓰고, 사용자가 읽고 승인 |
| Schema | **이 파일** + wiki-ingest/wiki-lint 스킬 | 위키 운영 규칙. AI와 co-evolve | 사용자 + AI |

## 3오퍼레이션

### Ingest (소스 → 위키 통합)
- **트리거**: 수동 호출 ("wiki-ingest"), 또는 다른 스킬에서 자동 호출
- **핵심**: 소스를 읽고 → 기존 페이지에 통합 → 핵심 갱신 → 교차 참조 강화 → 모순 감지
- **supervised mode**: 한 소스씩 처리, 사용자와 핵심 포인트 논의 후 반영 (기본)
- **batch mode**: 여러 소스 일괄 처리 (감독 최소화)

### Query (위키 검색 → 답변)
- **구현**: index.md 확인 → Grep → Read
- **트리거**: 사용자가 주제/개념에 대해 물으면 **00-wiki를 먼저 확인**
- **핵심**: 답변 결과가 가치 있으면 **다시 위키에 새 페이지로 저장**. 대화에서 나온 분석/비교가 위키에 남음

### Lint (위키 헬스체크)
- **트리거**: 수동 호출 ("wiki-lint", "위키 점검")
- **점검 항목**: 고아 페이지, 죽은 링크, 오래된 페이지(90일+), 모순, 누락 교차 참조, 비대한 페이지

## 복리의 메커니즘

단순 축적(파일 수 증가)이 아닌 **통합(integration)**:

1. **근거 추가** — 새 소스의 증거가 기존 페이지에 쌓임
2. **핵심 갱신** — 새 근거가 기존 이해를 바꾸면 핵심 요약이 더 정교해짐
3. **교차 참조 강화** — 같은 소스에서 여러 토픽이 enriched → 서로의 Related 연결
4. **모순 감지** — 새 내용이 기존 주장과 상충 → `[!contradiction]` 플래그
5. **Open Questions 해소** — 기존 질문이 새 소스로 답변되면 갱신
6. **Query 환류** — 대화에서 나온 분석/비교가 새 synthesis 페이지로 저장

## 페이지 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **concept** | 개념, 원칙, 아이디어 | 동사-중심-사고.md |
| **entity** | 특정 대상 (조직, 프로젝트, 도구) | Claude-Code.md |
| **synthesis** | 여러 소스의 비교/분석 | AI-도구-비교.md |

## 토픽 페이지 형식

```markdown
# [토픽명]

> **관련**: [[토픽A]] · [[토픽B]] · [[토픽C]]
> **Facets**: 키워드1 · 키워드2 · 키워드3

## 핵심 (현재 아는 것의 요약)
## 근거 (소스별 증거, [source: 출처, 날짜] 인용 필수)
## 적용 (내 맥락에서 실제 사용)
## Open Questions (아직 모르는 것)
## Related (다른 토픽 링크)
---
Sources: [소스 목록]
Last enriched: [날짜]
```

## 운영 규칙

- **기존 페이지 확장 우선**: 비슷한 토픽 있으면 새 파일 만들지 않음
- **source 인용 필수**: 모든 추가에 출처와 날짜
- **모순은 삭제하지 않음**: `[!contradiction]` 플래그만. 판단은 사용자
- **Query 환류**: 대화에서 나온 가치 있는 분석/비교는 synthesis 페이지로 저장
- **파일명**: 케밥케이스 (한글 또는 영어)
- **Co-evolution**: 이 SCHEMA 자체도 위키 운영 중 발견된 패턴을 반영하여 갱신
