# 트루스 디자인 시스템

> **관련**: [[trus|트루스]]
> **Facets**: 타이포그래피 · 비넬리 · 호흡 줄바꿈 · TRUS 팔레트 · 여백 · 한글 조판 · 버튼·모션·인터랙션

## 핵심 (한 줄 정의)

트루스 디자인 시스템은 **윤소정/트루스의 시각 언어** — 한 줄로 **"큰 글씨로 외치지 않고, 절제·호흡·여백으로 *읽히게* 한다."**

네 기둥:

1. **비넬리 타이포** — 소수의 크기만, 비율 1.25, **디스플레이도 본문의 2배 이내**. 강조는 *크기가 아니라 굵기·여백·위치*로.
2. **호흡 줄바꿈** — 소리 내어 읽어 *숨 쉬는 자리*에서 내려쓴다. **짧은 핵심 문장·리드에만**, 긴 설명 본문은 붙여서 흐르게.
3. **기능별 폰트** — 제목·라벨·짧은 선언 = **Pretendard** / 여러 줄 흐르는 본문 단락 = **명조(Noto Serif KR)**. 본문 **15px**.
4. **TRUS 팔레트** — BLACK `#121212` / WHITE `#FFFFFF` / **EDUCATE PINK `#D84D8D`**(시그니처, **5% 미만 절제**).

> **"트루스 디자인으로 해줘" = 이 시스템(특히 3색)을 적용한다.** 실행본: `trus-design.css` (link 한 줄로 전 페이지 재사용).

## 근거 (소스별 증거)

### 레퍼런스 3사 실제 CSS 분석
세 사이트의 production CSS를 직접 추출해 공통 원칙 도출. [source: design-guide.md §0, 2026-05-24]
- **Magazine B** — 여백 과잉형(섹션 간 100~128px), 위계를 크기·대문자로.
- **29CM** — Pretendard 단일, 조밀·정돈, 자간 0, 본문 14px.
- **Point of View** — 웜 에디토리얼, 영문 제목만 세리프, 섹션 80px.
- **공통 5원칙**: ① 폰트 1~2종 ② 위계는 굵기·여백(크기 대비 ❌) ③ 본문 자간 0 ④ 제목 줄간격 좁게·본문 넓게 ⑤ 여백 과감.

### 비넬리 원칙 (governing)
"You can say everything you want with very few type sizes." v1에서 디스플레이를 88px(`clamp 5.5rem`)까지 키운 게 비넬리 위반 → **31px 상한**으로 재정렬. 크기 비율 1.25. [source: 사용자 피드백, 2026-05-24]

### 호흡 줄바꿈 (윤소정식 핵심)
"잘 읽히게 하는 것"이 윤소정 타이포의 생명. ❌`"선생님이 수업 전에 / 하는 것"` → ✅`"선생님이 / 수업 전에 하는 것"`(주어 뒤 = 숨 쉬는 자리).
- **고아(orphan) 금지**: 조사(는/을/를)·단일 음절이 줄 끝에 혼자 떨어지면 안 됨. `word-break:keep-all` 필수. 안 되면 폰트 크기까지 조절해 완성도 유지.
- **적용 범위**: 짧은 핵심 문장·리드·제목·인용에만. 긴 "그냥 본문"은 붙여서 흐르게.
- **정렬**: 붙여 쓰는 본문 = 양쪽정렬(justify) + `word-break:normal`(빽빽한 블록, 책·신문식) / 짧은 문장 = 왼쪽정렬 + keep-all. [source: 사용자 피드백 세션, 2026-05-24]

### TRUS 공식 팔레트
사용자 제공 무드보드. 브랜드 서사 **틀에박힌(BOUND) → 깨어나는(AWAKENING) → 확장되는(EXPANDING)** — 핑크 = *각성·변화*. [source: 사용자 무드보드, 2026-05-24]

### Stripe Press 실 CSS 분석 — 웹·버튼·모션 (다섯 번째 기둥의 디테일)
`press.stripe.com`의 production CSS를 직접 추출. Stripe Press의 **태도가 TRUS와 동일** — 그림자 0·1px hairline·radius 2px·절제된 모션·채움↔외곽 반전. 폰트(Ivar 세리프 3종)·컬러(다크 #201819)는 버리고 **인터랙션 디자인 소스만 이식**. [source: press.stripe.com 실 CSS, 2026-05-25]
- **버튼** — 기존 TRUS 반전(채움↔외곽)과 동일. 추가 디테일: ① hover 이징 `cubic-bezier(.4,0,.2,1)` ② 키보드 포커스에만 링(클릭 땐 ❌) ③ `pointer:fine`=hover / `pointer:coarse`=`:active`로 분기(터치 hover 잔상 방지).
- **모션** — 이징 **딱 두 개**: 등장 `cubic-bezier(.16,1,.3,1)`, 전환 `cubic-bezier(.4,0,.2,1)`. hover = `opacity .65` 디밍이 공통 패턴. `prefers-reduced-motion` 가드 필수.
- **반전 카드** — BookZine CTA는 hover 시 *카드 통째로* 잉크 반전(`background:color; color:backgroundColor`). → `.card-interactive`.
- **시그니처** — ① 선택영역 색 반전(`::selection` 잉크↔화이트) ② hover에 미끄러져 들어오는 라벨(`translateX(-5px)→0`) ③ fluid 가장자리 여백 `calc(10px+1vw)` ④ 책표지 = 1px 액자 보더.
- **[!finding] 자간** — Stripe는 라벨/버튼에 **양수** 자간(.32px). TRUS CSS의 `.t-display -0.03em`·`.btn -0.01em`은 *"한글 음수 자간 ❌"* 규칙과 충돌 → Open Question 5로 이관(승인본 보호 위해 기존 값은 미수정).

## 적용 (내 맥락에서 실제 사용)

- **사이트 개편**: `identity.html`(승인본 정체성 페이지)이 `trus-design.css`를 import. → [[trus|트루스]] 브랜드의 시각 구현.
- **카드뉴스**: 부크크명조 + 비넬리 타이포 원칙으로 이미 제작 중 — 동일 기준.
- **핸드아웃(md-to-pdf)**: 호흡 줄바꿈·절제·여백 동일 적용.
- **재사용 한 줄**: `<link rel="stylesheet" href="trus-design.css">` → 본문 `.body-flow`, 버튼 `.btn-primary`, 핑크 강조 `.accent`.
- **웹·인터랙션 레이어 (2026-05-25 추가)**: `trus-design.css` 하단에 Stripe Press 학습본 추가. 모션 토큰(`--ease-out`/`--ease-hover`), `.btn` 포커스 링·pointer 분기, `.card-interactive`(잉크 반전), `.hover-label`(미끄러지는 라벨), `.framed`(액자 보더), `.rise`(등장). **눈으로 보는 데모**: `design/web-components.html`.

## Open Questions

1. ~~**버튼 모서리** — radius 2px vs 0?~~ ✅ **해소** — Stripe Press도 radius 2px·focus 2px. TRUS 기본값(2px) 확정. (2026-05-25)
2. **배경 톤** — 기본 TRUE WHITE. POV식 웜크림(#FAF6ED)은 특수 페이지만? 경계 미정. (Stripe는 다크 #201819 — TRUS는 채택 안 함, 화이트 유지)
3. **영문 디스플레이 세리프**(POV의 Larish 같은 것) — "그때그때 선택" 상태. 어떤 페이지에 쓸지 기준 필요.
4. **명조 본문의 장문 가독성** — 생각구독·ebook 같은 *진짜 장문*에서 명조 15px가 최적인가, Pretendard 본문이 나은가?
5. **음수 자간 충돌** — 규칙은 *"한글 음수 자간 ❌"*인데 `trus-design.css`의 `.t-display(-0.03em)`·`.t-l(-0.02em)`·`.btn(-0.01em)`이 음수. Stripe는 양수(.32px). 0/양수로 교정할지(승인본 영향) 사용자 판단 필요.
